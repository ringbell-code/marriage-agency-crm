## テストデータの動作

### ✅ **自動的に除外される場所**

1. **ダッシュボードのKPI**
   - 総会員数
   - 活動中会員数
   - 成婚退会数
   - 成婚率

2. **ダッシュボードの統計**
   - ステータス別集計
   - 面談未設定の会員
   - 30日以上連絡なし
   - 今週の面談予定

3. **会員一覧のデフォルト表示**
   - 通常は表示されません

### 👁️ **テストデータを表示する方法**

**会員一覧ページ:**
1. フィルターの「担当者」プルダウンを開く
2. 「テスト」を選択
3. テストデータのみが表示されます

**ダッシュボード:**
1. フィルターの「担当」プルダウンを開く
2. 「テスト」を選択
3. テストデータのみでKPIが計算されます

## テストデータの追加方法

### **Supabase SQL Editorから実行**

1. Supabaseにログイン
2. SQL Editorを開く
3. `supabase/add_test_members.sql` の内容を貼り付け
4. 「Run」をクリック

### **または、手動で追加**

会員一覧の「新規会員追加」ボタンから：
- **会員ID**: `TEST` で始まるID
- **氏名**: `テスト 〇〇`
- **担当者**: `テスト` を選択
- その他の情報を入力

## 注意事項

⚠️ **重要**: テストデータは本番データに影響を与えません
- 統計・KPIから自動的に除外されます
- 担当者フィルターで「テスト」を選択しない限り表示されません
- 削除しても問題ありません

## テストデータの削除方法

```sql
-- テストデータを削除
DELETE FROM members WHERE assigned_staff = 'テスト';
```

または、会員一覧で「テスト」でフィルタリングして、個別に削除してください。
